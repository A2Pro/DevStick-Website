Getting Started

Your DevStick comes pre-configured with your chosen Linux distribution and is ready to boot. The drive is configured for both UEFI and Legacy BIOS systems, ensuring maximum compatibility across different computers.

Important:

Before first boot, ensure Secure Boot is disabled in your BIOS settings. This is necessary for proper functionality of all Linux distributions.

Boot Instructions by System

Windows Systems

  1. 1. Plug in your DevStick while the computer is turned off
  2. 2. Power on your computer and immediately press the boot menu key:
    • • Dell: F12
    • • HP: F9
    • • Lenovo: F12 or Fn + F12
    • • ASUS: F8
    • • Acer: F12
    • • Microsoft Surface: Hold Volume Down
  3. 3. Select "USB Storage Device" or your DevStick's name from the boot menu

Mac Systems

  1. 1. Plug in your DevStick
  2. 2. Intel Macs:
    • • Restart and hold Option (⌥) key
    • • Select your DevStick from the boot menu
  3. 3. M1/M2 Macs:
    • • Shut down completely
    • • Hold the power button until "Loading startup options" appears
    • • Select your DevStick

Persistence Setup

Your DevStick comes with persistence enabled by default, meaning:

  • Your files and changes are saved between reboots
  • System updates are preserved
  • Installed applications remain available

Note:

The persistence partition is automatically mounted at boot. No additional configuration is required.

System Requirements

  • Dual-core CPU (2GHz+)
  • USB 2.0 port

Recommended

  • 4GB+ RAM
  • Quad-core CPU
  • USB 3.0 port

Troubleshooting

Common Issues

USB Not Detected

  1. 1. Try a different USB port
  2. 2. Ensure USB ports are enabled in BIOS
  3. 3. Check if USB boot is enabled in BIOS

Black Screen on Boot

  1. 1. Try booting with nomodeset:
    • • Press 'e' at boot menu
    • • Add 'nomodeset' after 'quiet splash'
    • • Press F10 to boot

WiFi Not Working

  1. 1. Open Terminal
  2. 2. Run: sudo service NetworkManager restart
  3. 3. If issue persists, check Additional Drivers menu
